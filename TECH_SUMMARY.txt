MyMoney Technical Summary

Product Overview

MyMoney is an offline-first personal finance app built with Expo and React Native.
It lets users manage accounts, transactions, budgets, categories, settings, and analytics.
The app stores data locally in SQLite, uses Expo Router for navigation, and Zustand for lightweight app state.

How The App Starts

The app boots through:
app/_layout.tsx

This file:
- shows the startup/loading state
- runs the bootstrap hook
- applies the navigation theme
- defines the root stack routes

Bootstrap logic lives in:
src/hooks/useBootstrap.ts

This hook:
- initializes the SQLite database
- loads persisted settings
- updates theme and currency values in the app store

Database initialization lives in:
src/db/migrations/init.ts

It:
- creates all database tables
- seeds default categories
- seeds default settings
- creates a default account
- ensures the hidden built-in "All Expenses" budget category exists

Low-level database access lives in:
src/db/database.ts

Architecture Pattern

Most features follow this flow:

screen -> hook -> service -> repository -> db query/schema

Meaning:
- screens render UI and call hooks
- hooks load data and manage loading/error/refresh behavior
- services contain business rules and validation
- repositories run SQLite reads and writes
- query files hold SQL
- schema files define tables

Feature Map

Transactions

Main files:
- screen list/filter UI: app/(tabs)/transactions.tsx
- create screen: app/add-transaction.tsx
- edit/delete screen: app/edit-transaction.tsx
- form UI: src/components/forms/TransactionForm.tsx
- hook: src/features/transactions/hooks/useTransactions.ts

- repository: src/features/transactions/repository/transactionRepository.ts

Important behavior:
- saving or deleting a transaction also updates the related account balance in the service layer

Budgets

Main files:
- budget tab: app/(tabs)/budget.tsx
- create/edit/delete screen: app/create-budget.tsx
- form UI: src/components/forms/BudgetForm.tsx
- hook: src/features/budgets/hooks/useBudgets.ts
- details hook: src/features/budgets/hooks/useBudgetDetails.ts

- repository: src/features/budgets/repository/budgetRepository.ts
- SQL: src/db/queries/budgetQueries.ts

Important behavior:
- budgets are unique by category + month + year
- the app supports a built-in special budget category called "All Expenses"
- the special category is defined in src/constants/categories.ts
- normal category lists hide that built-in row, but the budget system can still use it
- budget edit mode includes a delete action

Mental Model For New Developers

If you need to change:
- navigation or screen registration: start in app/
- screen data loading: check the feature hook
- business logic: check the feature service
- database reads/writes: check the feature repository and db query files
- shared UI/keyboard/layout behavior: check src/components/common/
- month/theme/currency/filter state: check src/store/

Quick File Routing Guide

If a user reports:
- transaction save issue -> transaction form, useTransactions, transactionService, transactionRepository
- balance mismatch -> transactionService and accountRepository
- budget amount/progress issue -> useBudgets, budgetService, budgetRepository, budgetQueries
- category list issue -> useCategories, categoryService, categoryRepository
- dashboard chart mismatch -> useDashboard, analyticsService, analyticsRepository
- theme or currency issue -> settings files, useBootstrap, appStore, theme files
- keyboard covering inputs -> ScreenContainer and TextField

Summary

This app is organized cleanly around feature slices.
Most day-to-day work should start by identifying the feature, then following the flow:
screen -> hook -> service -> repository -> query/schema

That pattern is the main key to understanding and extending the codebase safely.
